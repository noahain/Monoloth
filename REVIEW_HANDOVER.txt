1. WHAT WAS BUILT
--------------------------------------------------------------------------------

The CMD Panel (secondary terminal at the bottom of the main view, toggled
Ctrl+J, session_id = "panel") is now a multi-tab terminal. Each tab is an
independent PTY session with its own xterm.js renderer. Tab names default to
the shell type ("cmd" / "powershell" / "wsl" / etc.) and follow the existing
segmented-control look used by the settings tabs.

4.2  Exposed on window.SidebarManager (callable from app.js)
     createTab(name?, activate=true, dir?)  -> Promise<tab>
     activateTab(tabId)                      -> Promise<void>
     closeTab(tabId, force=false)
     getTab(tabId), getAllTabs(), getActiveTab(), getActiveTabId()
     getTabCount()

     initTabXterm(tab)                       (used by app.js restart flow)
     hideTabExitBanner(tab)
     isPanelOpen(), showCmdPanel(), hideCmdPanel(), toggleCmdPanel()

4.3  app.js changes
     - PTY output listener: if sessionId.startsWith('panel-tab-'), route to
       SidebarManager.writeToTab; existing 'panel' route is unchanged.
     - PTY exit listener: on 'panel-tab-*' exit, dispose term/fitAddon, clear
       bookkeeping, then re-init the xterm in-place (no new tab). Existing
       'panel' exit creates a new tab in its place.
     - New keyboard shortcuts (added to the existing global keydown handler):
         Ctrl/Cmd+T            open panel (if closed) + new tab
         Ctrl/Cmd+W            close active tab (only when panel open)
         Ctrl/Cmd+Tab          cycle to next tab (Shift for previous)
         Alt+1..9              jump to nth tab
     - On Ctrl+J (toggle panel open): if no tabs exist, create the first one.
     - On window close (existing beforeunload path): walks getAllTabs() and
       force-closes each, then calls hideCmdPanel().

4.4  Dirty-tracking (visual)
     When activating a tab, the previously-active tab gets `dirty=true` if
     it had received its first prompt (i.e. the user might have missed output
     while it was hidden). The red dot (●) is shown via
     `updateDirtyDot(tab)`. Dot is cleared on `closeTab` and on new output
     after re-activation (xterm write clears the dot).
     NOTE: the dirty dot is a hint, not a guarantee. xterm doesn't tell us
     when a background tab's content has actually changed beyond the
     firstPromptReceived gate.

4.5  Tab context menu (right-click)
     Built fresh per-open in showTabContextMenu(tabId, x, y). Listeners
     attached to the menu's items are removed on close + a delegated
     `document` contextmenu listener is removed when the menu hides
     (commit 004064d fixed a leak from the original implementation). Items:
     Rename, Duplicate, Close, Close Others, Close All.

4.6  IPC shape (unchanged contract)
     All calls go through window.monolithApi (set up in tauri-bridge.js with
     the { success, ... } wrapper — see AGENTS.md "bridge responses are
     wrapped"). No new IPC commands were added; tab sessions reuse
     start_terminal / send_input / resize_terminal / terminate_terminal.

8. WHAT TO REVIEW (please hit these explicitly)
--------------------------------------------------------------------------------

A. CONCURRENCY / RACE CONDITIONS
   - closeTab races with the PTY exit handler. If a tab is closed while
     a write is in flight, writeToTab may run on a disposed term. Check
     that term.write() inside writeToTab is guarded with `if (tab.term)`.
   - activateTab calls initTabXterm which calls start_terminal; the
     'panel-tab-N' generation is captured in _sessionGeneration[sessionId]
     in app.js. If two activateTab calls fire in quick succession (e.g.
     Ctrl+Tab mashing) you can get two start_terminal calls. Verify the
     second one no-ops or that the generation is bumped cleanly.

B. MEMORY / LISTENER LEAKS
   - On closeTab: term.dispose() and fitAddon.dispose() are called. Verify
     the ResizeObserver that watches the container is also torn down (it
     was added in initTabXterm for reflow). Grep for ResizeObserver in
     sidebar.js — confirm it disconnects on tab close.
   - The context-menu fix in 004064d removes the document-level listener
     after the menu hides. Confirm this is the ONLY document-level listener
     that showTabContextMenu installs; if there are more, they're still
     leaking.

C. ACCESSIBILITY
   - focus-visible is set on tab / + / × (cd0e08b). Is the visible
     outline distinguishable in CTA-blur and CTA-glass? (it uses
     --modal-text which is high-contrast on the panel surface).
   - Tab role / aria attributes: tabs are <div>s, not role="tab". If the
     bar is meant to be a tablist per WAI-ARIA, role="tablist" +
     role="tab" + aria-selected should be added. Confirm whether this is
     in scope.

D. SHORTCUTS
   - Ctrl/Cmd+T conflicts with "new tab" in many browsers — but this is
     a Tauri WebView, so it's only at risk if a user expects the WebView
     dev-tools shortcut. Confirm acceptable.
   - Ctrl/Cmd+Tab conflicts with the OS-level "switch app" / WebView
     focus cycling. May need to either guard it (only inside the panel)
     or rebind.

E. DIRTY DOT
   - Set on activate when firstPromptReceived is true. Cleared on new
     output after re-activation. Edge case: if user activates tab A, then
     output comes in for tab A and clears its dot — but the user already
     activated B in the meantime. The dot will clear on the next write,
     which is correct behavior, but it means the dot state can flicker.

F. SESSION CLEANUP ON CLOSE
   - lib.rs calls session_end_all_panel_tabs() and then terminate_all().
     PTYs are hard-killed by terminate_all, but the history records are
     written first. Verify the order in the actual on_window_event branch.

G. CSS TOKEN CONSISTENCY
   - --modal-active-bg is still defined in :root but no longer used by
     the tabs. Either repoint active to use it (for visual emphasis vs.
     hover) or remove the now-unused token. ponytail-review would say
     "delete it if nothing uses it".
   - --accent-warning is reused from the existing modal palette — confirm
     it actually exists in :root. If not, the dirty dot will render with
     no value and fall back to inherited color.

H. CACHE BUSTERS
   - All 11 bumped together. AGENTS.md says "bump ALL when you change
   any file". This is correct for the current changeset, but the values
   are now off-cycle from main (style.css is on v=125 on beta, v=123
   on main). When merging beta → main, main will need a separate
   cache-buster bump commit, or users updating from a main-2.0.5 build
   will pull stale JS. Flag this in the release notes.

I. TESTS
   - No new automated tests. The existing 22 cargo tests + 3 frontend
     vm-sandbox tests still pass (run `cargo test` and the three
     `node --test` commands listed in AGENTS.md). The renderer-policy
     sandbox does NOT exercise the new code paths — sidebar.js is
     too large to sandbox fully. Manual smoke test required.

J. KNOWN FOLLOW-UPS (not done, deliberately deferred)

9. HOW TO SMOKE TEST (suggested order, ~5 min)
--------------------------------------------------------------------------------

   1. `cargo tauri dev` — wait for window.
   2. Ctrl+J to open the panel. A first "cmd" tab should auto-create.
   3. Click +. A new tab should appear and be focused.
   4. Run `echo hi` in tab 1, switch to tab 2 (click), confirm tab 1
      shows the dirty dot ●.
   5. Switch back to tab 1. Dot should clear. Run `ping localhost`.
   6. Right-click tab 1 → Rename → "pinger". Tab label should update.
   7. Ctrl+W. Active tab closes. Repeat until empty — last remaining
      tab should NOT close.
   8. Ctrl+Tab to cycle, Alt+1/2/3 to jump. Confirm focus + xterm
      keyboard input land on the new active tab.
   9. Close the whole window via the X. Reopen. Open panel. The
      "panel" PTY should NOT have a phantom tab from the previous
      session (it's ephemeral — that's expected). Check the
      %APPDATA%/Monoloth/history.json to see the closed tab
      sessions are recorded as ended.

+    pub fn session_end_all_panel_tabs(&self) {
+        let mut inner = self.inner.lock();
+        let prefix = "panel-tab-";
+        let keys: Vec<String> = inner.active_sessions
+            .keys()
+            .filter(|k| k.starts_with(prefix))
+            .cloned()
+            .collect();
+        for key in &keys {
+            if let Some(active) = inner.active_sessions.remove(key) {
+                inner.data.sessions.push(SessionEntry {
+                    profile: active.profile,
+                    command: active.command,
+                    start_time: active.start_time,
+                    end_time: Some(iso_now()),
+                    directory: active.directory,
+                });
+            }
+        }
+        if !keys.is_empty() {
+            self.purge_inner(&mut inner);
+            save_json(&history_path(), &inner.data);
+        }
+    }

    function _doCloseTab(tabId) {
        var tab = _panelTabs.get(tabId);
        if (!tab) return;

        if (window.monolithApi) {
            window.monolithApi.terminate_terminal(tab.sessionId).catch(function () {});
        }
        if (tab.term) {
            try { tab.term.dispose(); } catch (e) {}
            try { tab.fitAddon.dispose(); } catch (e) {}
        }

        var tabItem = document.querySelector('.cmd-panel-tab[data-tab-id="' + tabId + '"]');
        if (tabItem) tabItem.remove();

        }

            var tabs = getAllTabs();
            var idx = tabs.findIndex(function (t) { return t.id === tabId; });
            var nextTab = tabs[idx - 1] || tabs[idx + 1] || tabs[0];
            if (nextTab) activateTab(nextTab.id);
        }
    }
